# Connecting the Interest Form to Odoo CRM

Your Odoo instance: `objetivo-excelencia-sl1.odoo.com` — CRM and Website apps are both installed, which is exactly what's needed.

## Why this approach

Odoo's Website app has a built-in "Form" block that can post directly to the CRM and create a real Opportunity — with Name, Email, and Phone landing in the correct fields (not just a text dump). To generate a working form, Odoo signs the exact set of fields you're allowed to submit with a security token (`website_form_signature`). That token can only be generated by Odoo itself, so the one-time setup below has you create a small form *inside* Odoo, then copy that token into the file I built for your site.

You'll only do this once. After that, every submission on your real website creates a proper Opportunity, with no further Odoo-side work per submission.

## One-time setup (~10 minutes)

**1. Go to your Website app** → `objetivo-excelencia-sl1.odoo.com/odoo/website`

**2. Create a new page** (Sitio → Nueva página). Call it something like `interest-form-source`. It never needs to be published or linked from anywhere — you'll only open it once to copy a value from it.

**3. Drag the "Formulario" (Form) building block** onto the page from the blocks panel.

**4. Click the form**, and in the right-hand settings panel find **Acción** (Action). Change it from the default "Enviar un correo electrónico" to **"Crear una oportunidad"** (Create an Opportunity). This is the key step — it rewires the form to create a `crm.lead` record instead of just sending an email.

**5. Leave the default fields as they are** (Nombre, Teléfono, Correo electrónico, Empresa, Asunto, Pregunta). They already match the field names I used in `interest-form.html`, so no extra configuration needed.

**6. Save the page** (Guardar). No need to publish it.

**7. Get the signature.** With the page open, right-click anywhere on the form → **"Ver código fuente de la página"** (View Page Source), or open DevTools (F12) → Elements tab. Search (Ctrl+F) for `website_form_signature`. Copy the long value inside `value="..."`.

**8. Open `interest-form.html`** (the file I delivered) in any text editor:
   - Find `PASTE_SIGNATURE_HERE` and replace it with the value you copied.
   - Find `YOUR-ODOO-INSTANCE.odoo.com` (there's one in a `<script>` near the bottom, and it's good practice to also set the `<form action="...">` at the top) and replace it with `objetivo-excelencia-sl1.odoo.com`.

**9. Upload `interest-form.html`** to the repo root on GitHub, same as any other page update. Netlify redeploys automatically.

**10. Test it.** Fill out the live form once yourself. Then check CRM → Flujo (Pipeline) → the "Nuevo" column. A new card should appear with your test name.

If nothing shows up: double-check the signature was copied in full (no line breaks), and that the Action really says "Crear una oportunidad" and not the default email option.

## What's already handled

All the extra questions that don't map to standard CRM fields — Montessori-info consent, what information they want, children's birth years, when they'd like to join, current school — get combined automatically into the lead's **Descripción** field as clean, readable text before submission. Nothing extra to configure for that.

## Next steps once leads are flowing in (for later)

These are separate, optional pieces of Odoo configuration — happy to help with any of them when you're ready:

- **Citas (Appointments) for campus visits**: once a lead requests a visit, staff can use the Citas app to send a booking link, or manually schedule directly from the Opportunity.
- **Marketing mailing list**: Odoo's Email Marketing app has Mailing Lists. Because "would like Montessori info" consent currently lives as text inside the Description (not a dedicated Yes/No field), automatically adding consenting leads to a list needs either a quick manual weekly glance, or a proper Boolean field added via Odoo Studio (paid add-on) for full automation.
- **Invoicing**: unrelated to this form — that happens later, once a family is actually enrolled, via the Sales/Invoicing apps.
